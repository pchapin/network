CIS-3152 Exam #1

Due: Friday, April 11, 2014

This exam covers all material up to and including email protocols.

  1. The call to connect in a typical sockets based TCP program uses a second argument that looks like (struct sockaddr *) &server_address where server_address is a struct sockaddr_in. Why is the type conversion necessary? In other words: why wasn't connect declared to take a struct sockaddr_in * in the first place?

  2. What is the advantage of using a concurrent TCP server? On the other hand, under what conditions would it be appropriate to use an iterative TCP server?

  3. Answer two of the three questions below.

    1. When a client program calls connect, what state or states in the TCP state diagram does the connection touch?
    2. What is the difference between "active close" and "passive close?"
    3. What sequence of states does the TCP connection touch when the side doing the passive close calls close? You can assume normal, error free behavior.

  4. UDP is an unreliable protocol. However, some services are most appropriately implemented as UDP services rather than using the more reliable TCP (DNS and TFTP are like this). Give at least one advantage that UDP has over TCP in these cases. Under what conditions is the unreliability of UDP not an issue?

  5. A program that uses UDP will use the sendto and recvfrom functions instead of the send/recv or write/read functions that a TCP based program would use. The sendto and recvfrom functions both take a pointer to a sockaddr structure as a parameter. What is the purpose of this parameter and why do the UDP functions require it while the TCP functions do not?

  6. An earlier version of the Pegasus Mail program allowed you to compose mail off-line. When you were finished writing the mail and clicked on "Send" the message was put into a queue for delivery when Pegasus Mail was once again connected to the network. However, the Date field on the outgoing message was set to when it was actually transmitted across the network; sometimes several hours after it was queued for delivery. Is this behavior correct? Justify your answer with appropriate reference to RFC-5322.

  7. What's the difference between a MIME Content-Transfer-Encoding of "8bit" and a Content-Transfer-Encoding of "binary?" Justify your answer with appropriate reference to RFC-2045.

  8. What is the base64 encoding of the UTF-8 string "=α="? What is the quoted printable encoding of the same string? Note that the character between the equals sign is the lower case greek letter alpha. You'll need to look up its Unicode code point (http://www.unicode.org) and then convert it to UTF-8 form to know exactly what binary bytes are to be encoded.

  9. Skim over RFC-1896 on the text/enriched MIME type. Use a text editor to manually construct a short message that demonstrates some of the features of enriched text. Don't forget to include appropriate RFC-5322 and MIME related header fields on your message so that it is a valid message.

  10. It is possible to send HTML mail (Content-Type: text/html) such that links to embedded images in the message refer to message attachments. This allows a person to email a complex document, complete with images, as an entirely self-contained message. Describe how this works (give references to appropriate documentation).

CIS-3152 Homework #8

This worksheet covers distributed applications using Ice. As a reference use your class notes and  the Ice documentation (which can be downloaded from http://www.zeroc.com).

  1. Suppose you wanted to implement a file server with an Ice interface. Using the slice interface definition language, write the specification for an interface File that allows read access to a file. The interface should include a close method as well. Also write the specification for an interface FileServer that has an open method returning a proxy to a File instance.

  2. Show C++ class definitions for classes File_impl and FileServer_impl that you might create to implement the interfaces described in the first question. You do not need to implement the methods themselves.

  3. On the Zeroc web site there are two documents comparing Ice with  CORBA and with SOAP (Web Services). Go to their site and pull down the "Ice" menu on the menu bar. Read those documents and write a short paragraph summarizing their main points.

  4. Ice (and similar technologies) involve allocating objects on various nodes in the network and passing references (called "proxies" in Ice) over the network to other nodes. Cleaning up allocated objects can be done explicitly or alternatively by using a distributed garbage collection algorithm. However, distributed garbage collection is hard (Ice doesn't do it). Why is it hard?
